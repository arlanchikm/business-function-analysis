# Бизнес-функция: бронирование отеля в сервисе туроператора

## Общие требования: 
Бизнес-функция бронирования отелей позволяет пользователю найти и выбрать подходящий отель по дате, местоположению, количеству гостей, а также забронировать подходящий номер в выбранном отеле и получить подтверждение бронирования.

## Цель функции.
Цель функции – предоставить пользователю возможность самостоятельного поиска и бронирования подходящего отеля по заданным им параметрам (город, дата проживания, количество гостей).

Функция служит для того, чтобы:
1.	Облегчить процесс поиска и оформления проживания и сократить его время на это.
2.	Обеспечить пользователю получение подтверждения бронирования без ручной обработки заявки менеджером
3.	Снизить нагрузку на менеджеров туроператора
4.	Увеличить долю онлайн бронирований

## Описание.
Функция охватывает процесс от ввода параметров поездки до получения пользователем подтверждения бронирования. Пользователь может указать город, дату проживания и количество гостей, после чего система выводит доступные для проживания варианты по заданным параметрам. 
Далее пользователь может применить фильтры (бюджет, количество комнат, условия проживания, количество звезд, рейтинг по отзывам, условия отмены бронирования, услуги отеля), выбрать подходящий вариант для проживания, ввести свои данные, оплатить бронирования жилья и получить подтверждение бронирования.

Система решает следующие задачи:
1.	помогает пользователю быстро найти подходящий отель
2.	показывает актуальные варианты размещения по заданным параметрам
3.	позволяет сравнить отели по заданному фильтру
4.	обеспечивает оформление бронирования без обязательного обращения к менеджеру
5.	фиксирует данные бронирования и передаёт их отелю
6.	предоставляет пользователю подтверждение бронирования

## Глоссарий.
Пользователь – посетитель сервиса
Гость – человек, который будет проживать в отеле
Номер – отдельное жилое помещение в отеле, предназначенное для временного проживания гостя
Бронирование – резервирование номера на выбранные латы
Ваучер – документ, подтверждающий бронирование

## Границы функции.

### Входит в функцию:
1.	поиск отеля
2.	фильтрация предложенных вариантов
3.	просмотр информации об отеле
4.	просмотр отзывов об отеле
5.	выбор номера в подходящем пользователю отеле
6.	ввод данных гостя
7.	cоздание бронирования
8.	получение подтверждения бронирования, ваучера
9.	передача данных бронирования отелю

### Не входит в функцию:
1.	покупка авиа- или ж/д билетов
2.	подбор пакетного тура
3.	бронирование экскурсии
4.	заказ трансфера

## Конкурентный анализ.
Были проанализированы следующие сервисы для бронирования жилья: ostrovok.ru, coral.ru, travel.yandex.ru, travelata.ru.
У конкурентов функция бронирования отеля строится по одинаковому сценарию: пользователь задаёт параметры поездки, получает список доступных отелей, уточняет выбор фильтрами, выбирает подходящий номер, вводит данные гостей, подтверждает условия и получает подтверждение бронирования, ваучер.

## Бизнес-цели
Основная бизнес-цель функции – обеспечить пользователю возможность самостоятельно забронировать отель онлайн.

### Бизнес-цели:
1.	увеличить количество онлайн-бронирований
2.	снизить нагрузку на менеджеров туроператора
3.	повысить конверсию из поиска отеля в бронирование
4.	сократить количество ошибок при оформлении бронирования
5.	ускорить и упростить получение подтверждения бронирования
6.	сохранить данные бронирования в личном кабинете пользователя

## User Story

### Поиск отеля
Как пользователь сервиса туроператора, я хочу найти отели по направлению, датам и количеству гостей, чтобы выбрать подходящий вариант проживания

### Фильтрация результатов
Как пользователь, я хочу фильтровать отели по цене, рейтингу по отзывам, количеству звезд, услугам отеля, условиям отмены бронирования, чтобы быстрее найти подходящий вариант

### Просмотр карточки отеля
Как пользователь, я хочу видеть описание отеля, фотографии, отзывы, и доступные номера, чтобы принять решение о бронировании

### Оформление бронирования
Как пользователь, я хочу ввести данные гостей и подтвердить условия, чтобы забронировать выбранный номер

### Получение подтверждения
Как пользователь, я хочу получить ваучер на email и видеть бронирование в личном кабинете, чтобы иметь подтверждение проживания

## Use Case
Название: бронирование отеля
Акторы: пользователь, платежная система
Цель: предоставить пользователю возможность самостоятельного поиска и бронирования подходящего отеля по заданным им параметрам
Предусловия: пользователь открыл сервис, есть доступные отели

### Основной сценарий
1.	пользователь задаёт параметры поиска
2.	система отображает доступные отели
3.	пользователь выбирает отель
4.	пользователь выбирает номер и тариф
5.	пользователь вводит данные гостей и контакты
6.	система проверяет корректность данных
7.	пользователь подтверждает условия бронирования
8.	система перенаправляет пользователя на страницу оплаты в платежной системе
9.	пользователь оплачивает бронирование
10.	 система создаёт бронирование
11.	 система получает подтверждение от поставщика
12.	 система отправляет ваучер пользователю

### Альтернативные пути
1.	Нет доступных отелей  
Система сообщает пользователю об отсутствии вариантов и предлагает изменить параметры поиска
2.	Номер стал недоступен  
Система сообщает, что выбранный номер недоступен, и предлагает выбрать другой вариант.
3.	Ошибка оплаты  
Система показывает сообщение об ошибке и предлагает повторить оплату или выбрать другой способ

Постусловия успешного бронирования: бронирование создано и подтверждено, ваучер отправлен пользователю.
Постусловия неуспешного бронирования: бронирование не создано, пользователь получает уведомление о причине и предложение попробовать снова.

## Функциональные и нефункциональные требования

### Функциональные требования
1.	система должна позволять пользователю искать отели по заданным параметрам (город, дата проживания, количество гостей)
2.	система должна отображать список доступных отелей с адресом, ценой и рейтингом
3.	система должна позволять фильтровать результаты по бюджету, количеству комнат, условиям проживания, количеству звезд, рейтингу по отзывам, условиям отмены бронирования, услугам отеля
4.	система должна позволять сортировать результаты по цене и рейтингу
5.	система должна отображать информацию о выбранном отеле (фотографии, описание, адрес, отзывы, доступные номера, услуги отеля)
6.	система должна позволять выбрать номер в выбранном отеле
7.	система должна отображать итоговую цену, условия отмены до подтверждения бронирования
8.	система должна позволять пользователю ввести данные гостей
9.	система должна перенаправлять пользователя на страницу оплаты в платежной системе и проверять статус платежа
10.	 система должна создавать бронирование после оплаты и отправлять информацию отелю
11.	 система должна отправлять ваучер пользователю по email
12.	 система должна отображать бронирование в личном кабинете пользователя
13.	 система должна позволять пользователю отменить бронирование в личном кабинете

### Нефункциональные требования
1.	система должна использовать HTTPS протокол
2.	система должна отображать результаты поиска не более чем через 3 секунды после нажатия пользователем кнопки «Поиск»
3.	дизайн системы должен поддерживать версию для слабовидящих
4.	система должна быть доступна не менее 99% времени в месяц
5.	пользователь не должен иметь доступ к бронированию других пользователей
6.	интерфейс системы должен корректно отображаться как на десктопных, так и на мобильных устройствах
7.	система должна хранить личные данные пользователей и гостей в соответствии с законодательством

## Ограничения
1.	дата выезда должна быть позже даты заезда
2.	бронирование должно быть доступно только при наличии свободных номеров
3.	пользователь должен указать данные гостей для бронирования
4.	бронирование должно подтверждаться только после оплаты пользователем
5.	ваучер должен отправляться пользователю только после подтверждения бронирования
6.	пользователь не должен иметь возможности бронировать номер на количество гостей, превышающее вместимость номера

4.	пользователь может применить фильтры и сортировку
5.	пользователь может посмотреть информацию об отеле
6.	пользователь может выбрать номер
7.	система отображает итоговую стоимость и условия отмены

9.	система создаёт бронирование

## Критерии приемки
1.	при вводе города, дат и количества гостей система отображает доступные отели
2.	если дата выезда раньше даты заезда, система показывает ошибку
3.	после выбора фильтра система показывает только подходящие отели
4.	система отображает информацию об отеле, включая фотографии, описание, адрес, отзывы, доступные номера и услуги отеля
5.	после выбора пользователем номера и заполнения данных гостей, система перенаправляет пользователя на страницу оплаты в платежной системе
6.	после оплаты система подтверждает бронирование, отображает его в личном кабинете пользователя и отправляет ваучер на email пользователя
